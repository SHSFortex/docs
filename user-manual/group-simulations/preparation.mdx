# Simulation Preparation

Administrators and facilitators use the Preparation phase to configure and launch group simulations.

## Creating a New Simulation

<Steps>
  <Step title="Start Creation">
    **Simulations** > **+ Create Simulation**
  </Step>

  <Step title="Select Scenario">
    Choose from library or create new scenario
  </Step>

  <Step title="Name the Simulation">
    Give it a descriptive name (e.g., "Q4 2024 Crisis Training - Marketing Team")
  </Step>

  <Step title="Configure Settings">
    Set parameters for this specific run
  </Step>

  <Step title="Add Participants">
    Invite team members
  </Step>

  <Step title="Schedule">
    Set start time or mark as "Start Manually"
  </Step>
</Steps>

## Configuration Options

### Basic Settings

- **Simulation Name**: Internal reference
- **Description**: Purpose and context
- **Start Date/Time**: When simulation begins
- **Duration**: Expected length (can extend if needed)
- **Time Zone**: For distributed teams

### Scenario Selection

Browse scenarios by:
- Category (Natural Disaster, Cyber Attack, etc.)
- Difficulty (Beginner, Intermediate, Advanced)
- Duration (15min, 30min, 1hr, 2hr+)
- Author (Your scenarios, Organization, Public)

### Participant Settings

<CardGroup cols={2}>
  <Card title="Role Assignment" icon="user-tag">
    **Manual**: Assign specific roles to participants
    **Automatic**: System distributes roles
    **Self-Select**: Participants choose roles
  </Card>
  <Card title="Team Size" icon="users">
    **Minimum**: Required participants
    **Maximum**: Participant limit
    **Optimal**: Recommended team size
  </Card>
</CardGroup>

## Adding Participants

### Individual Invitations

1. Click **+ Add Participant**
2. Search by name or email
3. Assign to role (if role-based scenario)
4. Set permissions (Observer, Active Participant)
5. Send invitation

### Bulk Invitations

1. Click **Bulk Invite**
2. Upload CSV with: Email, Name, Role
3. Review preview
4. Send all invitations

### Team Invitation

1. Select **Add Entire Team**
2. Choose from your teams
3. Auto-assign roles or manual assignment
4. Send invitations

<Note>
Participants receive email with simulation details, briefing link, and calendar invite.
</Note>

## Role Assignment

### Manual Assignment

For role-based scenarios:

<Steps>
  <Step title="View Roles">
    See all roles defined in scenario
  </Step>

  <Step title="Assign Participants">
    Drag participants to roles or use dropdown
  </Step>

  <Step title="Verify Coverage">
    Ensure all required roles filled
  </Step>

  <Step title="Backup Assignments">
    Assign backups for critical roles (optional)
  </Step>
</Steps>

### Auto-Assignment

Let the system assign roles:

1. Enable **Auto-Assign Roles**
2. System distributes participants based on:
   - Role requirements
   - Past experience (if available)
   - Balance across roles
3. Review and adjust if needed

## Simulation Permissions

### Participant Permissions

<Tabs>
  <Tab title="Active Participant">
    **Full Participation**

    - Make decisions
    - Communicate with team
    - Access all resources
    - Affects scoring
  </Tab>

  <Tab title="Observer">
    **View Only**

    - See events and decisions
    - Cannot vote or decide
    - Can view chat (read-only)
    - Doesn't affect scoring
  </Tab>

  <Tab title="Facilitator">
    **Control & Monitor**

    - All observer capabilities
    - Pause/resume simulation
    - Inject events
    - View admin dashboard
  </Tab>
</Tabs>

## Advanced Configuration

### Timing Controls

- **Auto-Start**: Begin at scheduled time
- **Manual Start**: Wait for facilitator to start
- **Auto-Advance Phases**: Move from Briefing → Execution → Debriefing automatically
- **Pause Allowed**: Can facilitator pause?

### Feature Toggles

<Check>**AI Mentor**: Enable AI guidance</Check>
<Check>**Team Chat**: Allow communication</Check>
<Check>**Resource Library**: Access to documents</Check>
<Check>**Timeline View**: Show event timeline</Check>
<Check>**Notes Feature**: Allow note-taking</Check>
<Check>**Real-Time Scoring**: Show live scores</Check>

### Difficulty Adjustments

Customize scenario difficulty:

- **Time Pressure**: Shorter/longer decision windows
- **Information Availability**: Limited or full resources
- **Event Frequency**: More/fewer timeline events
- **Hint System**: Enable/disable hints

## Pre-Simulation Checklist

<AccordionGroup>
  <Accordion title="24-48 Hours Before">
    - Scenario selected and configured
    - All participants invited
    - Roles assigned
    - Calendar invites sent
    - Briefing materials prepared
  </Accordion>

  <Accordion title="Day Before">
    - Send reminder email
    - Confirm participant availability
    - Test simulation in preview mode
    - Prepare debriefing questions
  </Accordion>

  <Accordion title="1 Hour Before">
    - Log in and verify simulation ready
    - Check all materials loaded
    - Test audio/video if applicable
    - Open facilitator dashboard
  </Accordion>

  <Accordion title="15 Minutes Before">
    - Participants can join briefing
    - Answer questions
    - Verify all participants present
    - Final tech check
  </Accordion>
</AccordionGroup>

## Testing the Simulation

### Preview Mode

Run through as a participant:

1. Click **Preview Simulation**
2. Experience briefing → execution → debriefing
3. Test decisions and branching
4. Verify timing
5. Check media loads correctly

### Dry Run

Full test with colleagues:

1. Invite test participants
2. Run abbreviated version
3. Identify technical issues
4. Refine timing
5. Adjust content as needed

## Managing Scheduled Simulations

### Viewing Upcoming Simulations

Dashboard shows:
- **Scheduled**: Future simulations
- **In Progress**: Currently running
- **Completed**: Finished simulations

### Editing Before Start

Can modify until simulation starts:
- Add/remove participants
- Change roles
- Adjust settings
- Reschedule start time

### Canceling Simulations

1. Open simulation
2. **Settings** > **Cancel Simulation**
3. Optionally notify participants
4. Confirm cancellation

## Troubleshooting

<AccordionGroup>
  <Accordion title="Participant can't join">
    - Verify they received invitation email
    - Check email address is correct
    - Ensure their email is verified
    - Resend invitation if needed
  </Accordion>

  <Accordion title="Not enough participants">
    - Send additional invitations
    - Use join code for last-minute additions
    - Adjust minimum participant requirement
    - Reschedule if necessary
  </Accordion>

  <Accordion title="Scenario not appearing">
    - Check scenario visibility settings
    - Verify it's published
    - Ensure you have access permissions
    - Refresh scenario library
  </Accordion>
</AccordionGroup>
