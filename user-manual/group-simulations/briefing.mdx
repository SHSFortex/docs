# Briefing Phase

The briefing phase prepares participants with context, instructions, and role assignments before the simulation begins.

## What is Briefing?

The briefing phase provides:

- **Scenario Context**: Background on the crisis situation
- **Learning Objectives**: What participants will practice
- **Role Assignments**: Individual responsibilities
- **Instructions**: How the simulation works
- **Ground Rules**: Expectations and guidelines

### Learning Objectives

Clear statement of what participants will:
- **Practice**: Skills being developed
- **Learn**: Knowledge to gain
- **Demonstrate**: Competencies to show

### Role Information

If role-based scenario, each participant sees:

- **Role Name**: Their position (e.g., "Crisis Manager")
- **Responsibilities**: What they're accountable for
- **Permissions**: What they can access/control
- **Team Members**: Others in the simulation and their roles

### Simulation Mechanics

Explanation of:
- **Duration**: How long it will run
- **Phases**: What to expect when
- **Features**: Chat, AI mentor, resources available
- **Scoring**: How performance is evaluated
- **Timeline**: When events will occur

## Pre-Simulation Activities

### Team Introduction

Facilitators may include:

**Ice Breaker**: Quick introductions
- Name
- Role in simulation
- Real-world role
- One expectation or concern

**Role Clarification**: Discuss responsibilities
- Questions about roles
- Coordination points
- Communication protocols

### Orientation

Walk through the interface:

<Tabs>
  <Tab title="Navigation">
    - Where to find timeline events
    - How to access resources
    - Using the chat feature
    - Decision-making interface
  </Tab>

  <Tab title="Features Demo">
    - Show AI mentor
    - Demonstrate decision process
    - Explain scoring (if visible)
    - How to take notes
  </Tab>

  <Tab title="Q&A">
    - Technical questions
    - Clarification on rules
    - Logistics
  </Tab>
</Tabs>

- **Attendance**: Who's joined, who's missing

- **Time Control**: Start countdown or begin immediately

## Best Practices

### For Facilitators

<AccordionGroup>
  <Accordion title="Start Briefing Early">
    Open briefing room 15 minutes before scheduled start. Allows time for stragglers and questions.
  </Accordion>

  <Accordion title="Set Expectations">
    Clarify:
    - Simulation is a learning environment
    - Mistakes are okay
    - Focus on teamwork
    - Stay in character
  </Accordion>

  <Accordion title="Confirm Understanding">
    Don't assume everyone read briefing materials. Summarize key points verbally.
  </Accordion>

  <Accordion title="Test Tech">
    Ask participants to send a test message in chat to verify functionality.
  </Accordion>

  <Accordion title="Build Engagement">
    Use the briefing to energize participants. Set a positive, professional tone.
  </Accordion>
</AccordionGroup>

### For Participants

<Check>**Arrive Early**: Join 5-10 minutes before start time</Check>
<Check>**Read Materials**: Review briefing before joining</Check>
<Check>**Ask Questions**: Clarify unclear points</Check>
<Check>**Test Features**: Send test chat message, explore interface</Check>
<Check>**Minimize Distractions**: Close unnecessary apps, find quiet space</Check>

## Common Questions

<AccordionGroup>
  <Accordion title="Can I change my role?">
    Contact the facilitator during briefing. They may reassign roles if necessary.
  </Accordion>

  <Accordion title="What if I join late?">
    You can join during briefing. If you join after execution starts, you may enter in-progress.
  </Accordion>

  <Accordion title="Can I leave during the simulation?">
    Avoid leaving if possible, but if necessary, notify your team and facilitator.
  </Accordion>

  <Accordion title="Will I get a copy of briefing materials?">
    Yes, briefing and resources typically remain accessible during execution.
  </Accordion>

  <Accordion title="How long does briefing last?">
    Usually 10-15 minutes, but depends on scenario complexity and team familiarity.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't see briefing content">
    - Refresh browser
    - Check internet connection
    - Verify you clicked correct link
    - Clear cache and try again
  </Accordion>

  <Accordion title="Facilitator not present">
    - Wait a few minutes
    - Check simulation start time
    - Use chat to communicate with other participants
    - Contact support if prolonged absence
  </Accordion>

  <Accordion title="Don't understand my role">
    - Ask in chat
    - Request facilitator explain
    - Review role description in briefing
    - Coordinate with team members
  </Accordion>
</AccordionGroup>
