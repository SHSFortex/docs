# Kommunikationsfunktionen

Effektive Kommunikation ist im Krisenmanagement entscheidend. crisisIQ bietet Tools für die Teamzusammenarbeit während Simulationen.

### Chat-Funktionen

- **Echtzeit-Synchronisation**: Nachrichten erscheinen sofort
- **@Erwähnungen**: Spezifische Teammitglieder markieren
- **Reaktionen**: Schnelle Emoji-Antworten
- **Dateifreigabe**: Screenshots, Notizen teilen
- **Suche**: Frühere Nachrichten finden
- **Benachrichtigungen**: Warnung bei neuen Nachrichten

## Effektive Krisenkommunikation

### Kommunikationsprinzipien

<Tabs>
  <Tab title="Klarheit">
    **Klar und prägnant sein**

    - Fakten deutlich angeben
    - Fachjargon wenn möglich vermeiden
    - Spezifizieren, wer was tun soll
    - Aufzählungspunkte für Listen verwenden
  </Tab>

  <Tab title="Rechtzeitigkeit">
    **Schnell kommunizieren**

    - Kritische Infos sofort teilen
    - Nicht auf perfekte Informationen warten
    - Regelmäßig aktualisieren
    - Empfang von Nachrichten bestätigen
  </Tab>

  <Tab title="Relevanz">
    **Teilen, was wichtig ist**

    - Signal vom Rauschen filtern
    - Dringende Punkte hervorheben
    - Kontext liefern
    - Geeignete Personen markieren
  </Tab>

  <Tab title="Zweiseitig">
    **Zuhören und antworten**

    - Um Input bitten
    - Nachrichten anderer bestätigen
    - Fragen beantworten
    - Verständnis bestätigen
  </Tab>
</Tabs>

### Nachrichtenvorlagen

**Status-Update:**
```
UPDATE: [Thema]
Status: [Aktuelle Situation]
Aktion: [Was wir tun]
Bedarf: [Erforderliche Unterstützung/Ressourcen]
```

**Entscheidungsanfrage:**
```
ENTSCHEIDUNG ERFORDERLICH: [Problem]
Optionen: [A, B, C]
Frist: [Wann wir entscheiden müssen]
Input: [Welches Feedback benötigt]
```

**Informationsweitergabe:**
```
INFO: [Betreff]
Quelle: [Woher dies kam]
Details: [Schlüsselinformationen]
Auswirkung: [Wie dies uns betrifft]
```

**Aufgabenzuweisung:**
```
AKTION für @Name:
Aufgabe: [Was getan werden muss]
Warum: [Kontext/Grund]
Bis wann: [Frist]
```

## Externe Stakeholder-Kommunikation

### Kommunikationsentscheidungen

Während Simulationen müssen Sie möglicherweise entscheiden:

- **Wann kommuniziert werden soll**: Sofort vs. auf weitere Infos warten
- **Was zu sagen ist**: Vollständige Offenlegung vs. begrenzte Infos
- **Wen zu informieren ist**: Welche Stakeholder zuerst
- **Wie zu kommunizieren ist**: Pressemitteilung, E-Mail, soziale Medien
- **Wer spricht**: CEO, PR-Team, technischer Experte

## Kommunikationsherausforderungen

### Informationsüberflutung

**Problem**: Zu viele Nachrichten zum Verarbeiten

**Lösungen**:
- @Erwähnungen und dringende Markierungen priorisieren
- Nach rollenspezifischen Infos scannen
- Suche verwenden, um relevante Nachrichten zu finden
- Periodisch zusammenfassen

### Koordinationslücken

**Problem**: Teammitglieder nicht abgestimmt

**Lösungen**:
- Regelmäßige Status-Check-ins
- Explizite Entscheidungsbestätigungen
- Wiederholung für kritische Infos
- Klare Aktionsverantwortliche zuweisen

### Kommunikationsausfall

**Problem**: Nachrichten kommen nicht an

**Lösungen**:
- Prüfen, ob Empfänger aktiv sind
- Mehrere Kanäle verwenden, wenn dringend
- Eskalieren, wenn keine Antwort
- Moderator bei technischem Problem alarmieren

## Kommunikationseffektivität überwachen

### Selbstbewertungsfragen

Während der Simulation:
- Nehmen alle am Chat teil?
- Sind Nachrichten klar und umsetzbar?
- Werden kritische Informationen geteilt?
- Koordinieren wir effektiv?
- Wird jemand aus der Kommunikationsschleife ausgeschlossen?

Nach der Simulation:
- Chat-Protokolle auf Muster überprüfen
- Kommunikationslücken identifizieren
- Notieren, was gut funktioniert hat
- Verbesserungen für das nächste Mal planen
